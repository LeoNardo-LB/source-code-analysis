## Goals / Non-Goals

**Goals:**
- 修复所有遗留的事实性错误（ER 图缺字段、行号偏差、签名不完整等）
- 去重 6 处跨文档重复内容，改为交叉引用
- 补充遗漏的重要源码模块描述
- 统一数量/路径等引用

**Non-Goals:**
- 不重写文档内容风格或措辞
- 不新增文档文件（不创建新 .md）
- 不修改文档目录结构（文件不移动/重命名）
- 不修改源码（纯文档变更）
- 不删除任何 Mermaid 图表（去重时用引用替代）
- 不修改附录文件（A/B/C）

## Decisions

### D1: 去重策略 — 简化 + 引用而非删除

**选择**：在非权威文档中，将重复内容简化为摘要段落 + `详见 [权威文档](路径)` 链接。

**替代方案**：
- A) 完全删除重复内容，只留引用 → 阅读体验差，跳转太多
- B) 保持重复但标注"与 XX 文档保持同步" → 维护负担仍在

**理由**：摘要 + 引用平衡了可读性和维护性。读者在当前文档中能看到核心要点，需要细节时跳转。

### D2: 权威文档分配

| 知识点 | 权威文档 | 其他文档处理 |
|--------|----------|-------------|
| 系统架构/技术栈/包结构 | 00-总览 | README 简化引用 |
| 数据模型/ER 图/存储路径/PRAGMA | 15-存储 | 02-Session 简化引用 |
| runLoop 执行循环 | 03-Agent | 02-Session 简化引用 |
| Provider 接口/注册 | 04-Provider | 12-自定义Provider 简化引用 |
| 权限检查流程 | 09-权限控制 | 06-文件操作 简化引用 |

### D3: ER 图处理 — 分层保留

**选择**：02-Session 保留精简版 ER 图（仅 Session/Message/Part/Todo 四个实体），15-存储 保留完整版（含 Project/Permission/SessionShare/Account）。

**理由**：02 的读者关注 Session 数据流，不需要知道 Project 或 Permission 的字段。

### D4: 遗漏模块补充方式

**选择**：在 00-总览的核心模块一览表中补充 5 个重要遗漏模块（auth/、control-plane/、project/、env/、command/），其他次要模块不补充。

**理由**：全部 44 个模块列出来会过于冗长。5 个是开发者最可能需要了解的。

## Risks / Trade-offs

- **[风险] 去重后单文档信息密度增加** → 通过摘要段落缓解，确保读者不跳转也能理解核心
- **[风险] 交叉引用路径可能随文件重命名失效** → 本次不重命名文件，且在 Non-Goals 中约束未来
- **[风险] 源码行号随版本更新失效** → 这是固有问题，本次修正当前版本行号，无法根治
- **[取舍] ER 图去重 vs 完整性** → 选择分层保留而非完全合并，牺牲少量重复换取可读性
